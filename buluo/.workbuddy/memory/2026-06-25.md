# 2026-06-25 工作日志

## CoC 升级提醒平台 - JSON 解析器修复

### 问题
- `building_map.json` 的 SC ID 映射完全错误 (buildings 用了 1000001 起, 但实际 Town Hall 是 1000001, Cannon 是 1000008)
- spells 用了 2000xxx 前缀, 实际应为 26000xxx
- heroes 用了 3000xxx, 实际应为 28000xxx
- pets 用了 4000xxx, 实际应为 73000xxx
- equipment 用了 5000xxx, 实际应为 90000xxx
- 解析器缺少 `helpers` 类别支持 (用 `helper_cooldown` 而非 `timer`)
- `buildings` 和 `buildings2` 共享同一 ID 空间 (1000xxxx), 扁平化映射表导致后者覆盖前者
- `next.config.js` 缺少 API rewrites 配置, 前端请求无法代理到后端

### 修复
1. **building_map.json** - 使用真实 CoC SC ID 重写全部映射 (10 个类别, 122 个 ID):
   - buildings (1000xxxx): 35 个, 含 Laboratory(1000007), Barracks(1000006), Eagle Artillery(1000031) 等
   - spells (26000xxx): 14 个
   - heroes (28000xxx): 5 个, 含 Minion Prince(28000002)
   - pets (73000xxx): 11 个
   - equipment (90000xxx): 13 个
   - units (4000xxx): 21 个
   - helpers (124000xxx): 3 个
   - buildings2/heroes2/units2: 夜世界类别

2. **coc_parser.py** - 改为按 category 分层查找 ID 映射, 解决 buildings/buildings2 ID 冲突:
   - `load_id_map()` 返回嵌套 dict `{category: {id: info}}`
   - `resolve_name()` 先精确 category 查找, 再跨 category 搜索
   - 新增 `_parse_helpers()` 处理 `helper_cooldown` 字段
   - 支持所有类别: buildings, spells, heroes, pets, equipment, units, helpers, buildings2, heroes2, units2, siege_machines, traps, traps2 等

3. **models.py** - Upgrade 表新增 `timer_seconds` 列
4. **schemas.py** - `UpgradeResponse.timer_seconds` 改为 `int | None = None`
5. **api.py** - 上传时保存 `timer_seconds` 到数据库
6. **scheduler/tasks.py** - 通知模板新增所有类别的中文映射
7. **next.config.js** - 添加 rewrites 将 `/api/*` 代理到 `http://localhost:8000/api/*`
8. **frontend/types/index.ts** - CATEGORY_MAP/ICON/COLOR 新增 helpers, units, buildings2 等所有类别

### 验证结果
- 解析器测试: 8 个升级项全部正确识别 (Laboratory, Barracks, Eagle Artillery, Pet House, Hero Hall, Builder Hut, Bat Spell, Helper 1)
- API 测试: POST /api/upload-json 返回正确 JSON, 名称/等级/倒计时均正确
- 前后端代理: Next.js (3000) → FastAPI (8000) 链路打通

## 第四轮: PWA 升级 (SaaS 级重构)

### 核心改造
1. **manifest.json** — PWA 配置: name/description/standalone/theme_color/icons
2. **public/sw.js** — Service Worker: CacheFirst 静态资源 / NetworkFirst API / Push通知 / BackgroundSync
3. **lib/coc-assets.ts** — 完整 CoC 中文映射: 17 个分类表 (~200+ 条), 含中文名/英文名/图标emoji/颜色, 自动 fallback 按 ID 前缀匹配
4. **lib/indexeddb.ts** — IndexedDB 存储层: 升级数据/用户数据/历史/通知配置 4 个对象存储, 替代 localStorage
5. **lib/notification-system.ts** — 通知管理器: WxPusher + 浏览器 Notification + PWA Push 三通道, 自动安排升级完成时间通知
6. **hooks/usePwaInstall.ts** — PWA 安装钩子: 检测 beforeinstallprompt, 拦截安装, 判断 standalone/installed 状态
7. **app/page.tsx** — 全面重构: Hero 区域(玩家信息) / 最近即将完成高亮卡 / 实时倒计时(每秒) / 分类颜色卡片 / 中文显示 / 通知设置面板 / JSON Diff / 数据过期提醒 / 移动端适配
8. **app/layout.tsx** — 注入 manifest 链接 + apple-mobile-web-app meta + PWA icons
9. **lib/utils.ts** — 增强: getCategoryIcon/getCategoryBg/getCategoryBorder/getCategoryText/getCategoryBadge, 调用 coc-assets 中文映射
10. **app/globals.css** — 新增: tabular-nums倒计时对齐 / reduced-motion无障碍 / iOS 16px 输入防缩放
11. **next.config.js** — 保留 API proxy, manifest.json 和 sw.js 通过 public/ 自动服务

### 技术亮点
- 纯前端 PWA 方案, 不再需要后端 FastAPI 提供 UI (但后端仍保留作 API)
- 中文名称三级 fallback: data_id 精确匹配 → ID 前缀范围匹配 → category 通用映射
- 倒计时使用 react tick (setInterval 1s) 驱动 DOM 重渲染, 配合 tabular-nums 数字不对齐
- IndexedDB 在上传时自动持久化升级数据, 页面刷新后从 IDB 恢复
- Service Worker 注册在 layout.tsx, 自动缓存 public/ 下的静态资源
